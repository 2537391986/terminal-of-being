# 《存在终端》版本规划 v2.0

> **定位**：从"能玩"到"好玩"的路线图。按版本迭代，每个版本有明确目标和可验收标准。
> **原则**：不做完美规划，做最小可验证版本。每版完成后玩家能感受到明显变化。

---

## 当前状态（v0.7 — Boss 有灵魂 ✓）

- ✅ 50 把武器，每把有哲学概念介绍
- ✅ 终端风格 UI

- ✅ **Boss 阶段系统完整实现**（5个Boss × 3~4阶段，含视觉特效）

## v0.7 — 「Boss 有灵魂」✅ 已完成

### 目标
让 5 个 Boss 战成为玩家的记忆点，而不只是"血厚的怪"。

### 具体任务

#### 1. Boss 阶段系统（核心）
- [x] `BOSS_PHASE_DATA` 数据结构已定义（enemies.js）
- [x] 阶段转换触发逻辑（`checkBossPhase()` in combat.js:287）
- [x] 阶段转换视觉反馈（全屏震动+慢动作+粒子爆发+飘字）
- [x] 5 个 Boss 各 3~4 个阶段，每阶段有独特机制：

| Boss | 阶段机制 | 实装效果 |
|------|---------|---------|
| 此在残响 | 攻速渐增（1.35x → 1.7x → 2.2x） | aspd 叠加 + crit 提升 |
| 强力意志化身 | 每次受击攻击力永久上升 | onHitAtkGain 叠层 |
| 物自体防壁 | 伤害吸收递增（20% → 45% → 65%） | damageShieldPct 递增 + defMul |
| 明希豪森之结 | 假死复活3次+召唤残影 | fakeDeath + summonOnDeath + healPct |
| 统觉统一场 | 伤害合并窗口/忽略小额伤害/仅最大一击 | mergeWindow + ignoreChipDamage + onlyLargestHit |

#### 2. Boss 战 UI 增强
- [x] Boss 血条阶段标记（renderer.js:289-297 — 竖线标记各阶段阈值）
- [x] 阶段转换时屏幕震动+慢动作+粒子爆发（combat.js:331-357）
- [x] 阶段名称飘字显示（红字浮在Boss上方）
- [x] 当前阶段名称显示在血条右侧（renderer.js:301-308）

#### 3. 验收标准
- [ ] 打 20 层 Boss，能看到至少 2 次阶段转换（需实际跑一遍验证）
- [ ] 每个 Boss 的阶段机制让玩家必须改变策略（需实际跑一遍验证）
- [ ] 无 JS 报错（需实际跑一遍验证）

## v0.8 — 「经济有闭环」（预计 3~4 天）

### 目标
金币和装备有来源也有消耗，玩家需要做选择。

### 具体任务

#### 1. 金币消耗出口
- [ ] **概念商店**（codex 面板里）：花金币解锁概念条目（每个 100~500 金）
- [ ] **装备分解**：白/绿装分解成「概念残渣」，残渣可用于：
  - 在概念商店「兑换」指定流派的装备（稀有度越高越贵）
- [ ] **装备重铸**（可选）：消耗金币 + 残渣，随机刷新装备的词缀

#### 2. 掉落经济校准
- [ ] 写一个小脚本 `simulate-drop.js`，跑 1000 次击杀模拟，输出：
  - 各稀有度实际掉率
  - 玩家到 50 层平均有几件史诗/传说
  - Boss 掉率是否过高/过低
- [ ] 根据模拟结果调整 `dropWeight`

#### 3. 验收标准
- 金币不会无限积累（有地方花）
- 白装不再是纯垃圾（可分解）
- 模拟报告显示稀有度分布符合设计文档（普通 60% / 稀有 30% / 史诗 8% / 传说 2%）

## v0.9 — 「世界有广度」（预计 5~7 天）

### 目标
 monster 池扩容，让每 10 层都有新东西。

### 具体任务

#### 1. 怪物池扩容（17 → 40+）
新增怪物覆盖：
- 道家（道、无为、阴阳）
- 佛教（四圣谛、八正道、缘起）
- 实用主义（詹姆斯、杜威）
- 科学哲学（波普尔、库恩、费耶阿本德）
- 女性主义（波伏娃、格雷塔·通贝里）
- 政治哲学（阿伦特、罗尔斯）

每个新怪物对应一把新武器（或反之）。

#### 2. 敌人 Variety 增强
- [ ] 怪物不再只是「站桩对打」，加入：
  - **远程怪**：投射物攻击（需要在 renderer.js 画子弹）
  - **高速怪**：移动速度快但血量低
  - **坦克怪**：移动慢但护甲高

#### 3. 关卡进度感知
- [ ] 每 10 层一个「检查点」，显示「第 N 章：[章节名]」（如「第三章：理性的边界」）
- [ ] 章节名与当期 Boss 的哲学流派对应

#### 4. 验收标准
- 打到第 50 层，遇到的怪物种类 ≥ 30
- 新怪物有新攻击模式（不只是数值不同）
- 玩家能感受到「章节感」

## v1.0 — 「可以发布了」（预计 7~10 天）

### 目标
 polish，让游戏「完整」。

### 具体任务

#### 1. 新手引导
- [ ] 第一次打开游戏，显示 3 步引导：
  1. 「这是你的终端——点击屏幕开始涌现」
  2. 「击败敌人获得装备——点击装备查看哲学概念」
  3. 「图鉴记录你遇到的概念——点击右上角按钮打开」
- [ ] 引导只显示一次（用 localStorage 记录）

#### 2. 死亡体验
- [ ] 死亡时不直接重置，显示「存在崩解」1.5 秒动画
- [ ] 显示本局统计：击杀数 / 最高层数 / 获得装备数 / 解锁概念数
- [ ] 「重新抛入」按钮（回到上一层）

#### 3. 音效（可选，如果时间够）
- [ ] 用 Web Audio API 生成简单的 8-bit 风格音效：
  - 攻击命中：短促的「哔」
  - 暴击：两声「哔哔」音调更高
  - 掉落装备：上升音阶「哔哔哔」
  - Boss 登场：低沉的「嗡————」

#### 4. 性能优化
- [ ] Canvas 粒子数量上限（避免 100 个伤害数字同时显示卡顿）
- [ ] 怪物数量上限（同一屏最多 5 个）
- [ ] FPS 监控（右下角小字显示）

#### 5. 文档 & 发布准备
- [ ] 更新 `GAME_DESIGN.md` 与实际代码对齐
- [ ] 写 `README.md`（玩法简介 + 本地运行方法 + 设计理念）
- [ ] GitHub Pages 部署成功（https://2537391986.github.io/terminal-of-being/）
- [ ] 写一段 140 字以内的「电梯演讲」（用于分享链接时显示）

#### 6. 验收标准
- 新玩家 30 秒内知道要做什么
- 死亡后有「再来一把」的动力
- 在 GitHub Pages 上跑 10 分钟无崩溃
- README 能让一个不懂哲学的人也想玩
